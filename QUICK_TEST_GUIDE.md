# Быстрый тест перед конкурсом

## 🚀 Запуск

```bash
# 1. Backend
cd backend
docker-compose up -d
uvicorn app.main:app --reload

# 2. Frontend
cd frontend
npm run dev
```

## ✅ Что протестировать (5-10 минут)

### 1. Уведомления (КРИТИЧНО)

```bash
# Сценарий:
1. Войти как user@test.com / user123
2. Создать проблему
3. Войти как другой пользователь (moderator@test.com / moderator123)
4. Добавить комментарий к проблеме
5. Вернуться к user@test.com
6. Проверить колокольчик уведомлений в Header
7. Открыть /notifications - должно быть уведомление о комментарии

✅ Ожидаемый результат: Уведомление появилось
```

### 2. Moderator Panel

```bash
# Войти как moderator@test.com / moderator123
1. Открыть /admin (кнопка Shield в Sidebar)
2. Перейти на вкладку "Moderator"
3. Проверить:
   - Жалобы на комментарии (если есть)
   - Подозрительные проблемы (низкий truth_score)
   - Новые проблемы
4. Попробовать верифицировать проблему
5. Проверить что автор получил уведомление

✅ Ожидаемый результат: Панель работает, уведомления приходят
```

### 3. Official Panel

```bash

✅ Ожидаемый результат: Workflow работает, уведомления приходят
```

### 4. Admin Panel


## 🐛 Если что-то не работает

### Уведомления не приходят

```bash
# Проверить логи backend:
tail -f backend/logs/app.log

# Проверить что notification_service импортируется:
cd backend
python -c "from app.services.notification_service import NotificationService; print('OK')"
```

### Тестовые аккаунты не создались

```bash
# Проверить .env:
cat backend/.env | grep CREATE_TEST_ACCOUNTS

# Должно быть:
CREATE_TEST_ACCOUNTS=true

# Пересоздать:
cd backend
docker-compose down
docker-compose up -d
uvicorn app.main:app --reload
```

### Frontend не подключается к backend

```bash
# Проверить что backend запущен:
curl http://localhost:8000/api/v1/health

# Проверить VITE_API_URL в frontend/.env:
VITE_API_URL=http://localhost:8000
```

## 🔥 Что подчеркнуть жюри
