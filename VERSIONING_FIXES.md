## 🔥 Проблемы с версионированием

### 1. created_at обновляется при версионировании ✅ ИСПРАВЛЕНО

**Проблема**: 
- При создании новой версии проблемы `created_at` обновлялся
- Проблема показывала "0 дней не решена" после добавления комментария
- В `versioning.py` строка 60 пропускала `created_at` в skip

**Решение**:
```python
# Было:
skip = {"id", "is_current", "created_at", "superseded_at"}

# Стало:
skip = {"id", "is_current", "superseded_at"}
# created_at теперь копируется из первой версии!
```

**Файл**: `backend/app/services/versioning.py`

### 2. Модерация новых проблем ✅ ДОБАВЛЕНО

**Проблема**: 
- Любой пользователь мог создавать проблемы без модерации
- Проблемы сразу появлялись на карте

**Решение**:
1. Добавлен статус `pending` в `ProblemStatus`
2. Новые проблемы создаются со статусом `pending`
3. Модератор/админ должен одобрить через `/moderator/problems/{id}/verify`
4. После одобрения статус меняется на `open`

**Изменения**:
- `backend/app/models/problem.py` - добавлен статус `pending`
- `backend/app/api/v1/problems.py` - новые проблемы создаются с `status=pending`
- `backend/app/api/v1/moderator.py`:
  - `/problems/pending` - показывает проблемы со статусом `pending`
  - `/problems/{id}/verify` - меняет статус на `open`

### 3. Reports и несуществующие проблемы ⚠️ ТРЕБУЕТ ПРОВЕРКИ

**Проблема**:
- В ModeratorPanel ссылки на проблемы ведут на несуществующие ID
- Используется `target_entity_id` для ссылок

**Анализ**:
- Reports правильно хранит `target_entity_id` (это entity_id проблемы)
- Ссылки используют `/problems/${target_entity_id}` - это ПРАВИЛЬНО
- Проблема скорее всего в том, что:
  1. В БД есть reports на удаленные/отклоненные проблемы
  2. Или reports созданы с неправильным target_entity_id

**Что проверить**:
```sql
-- Найти reports с несуществующими проблемами
SELECT r.entity_id, r.target_entity_id, r.target_type
FROM reports r
WHERE r.is_current = true 
  AND r.target_type = 'problem'
  AND NOT EXISTS (
    SELECT 1 FROM problems p 
    WHERE p.entity_id = r.target_entity_id 
      AND p.is_current = true
  );
```

**Возможное решение**:
1. Добавить проверку в Reports API - не показывать reports на несуществующие объекты
2. Или добавить каскадное удаление - при удалении проблемы помечать связанные reports как resolved
