# Система оценки проблем (Scoring System)

## Обзор

CityProblemMap использует систему взвешенных оценок для определения достоверности и приоритета городских проблем. Все оценки пересчитываются автоматически после каждого голоса.

---

## 📊 Метрики

### 1. Достоверность (Truth Score)

**Диапазон**: 0-100%  
**Назначение**: Определить, является ли проблема реальной или фейком

#### Формула

```
truth_score = (сумма весов голосов "Правда") / (сумма весов всех голосов)
```

#### Как работает

- Учитываются только голоса с явным выбором "Правда" или "Фейк"
- Воздержавшиеся (не выбравшие is_true) не влияют на достоверность
- Каждый голос взвешивается по репутации пользователя

#### Веса голосов

| Репутация пользователя | Вес голоса |
|------------------------|------------|
| Новичок (0-100)        | 1.0        |
| Активный (100-500)     | 1.5        |
| Опытный (500-1000)     | 2.0        |
| Эксперт (1000+)        | 2.5        |

#### Пример расчёта

**Голоса**:
- 3 новичка → "Правда" (вес 1.0 каждый) = 3.0
- 1 опытный → "Фейк" (вес 2.5) = 2.5
- 2 новичка → "Правда" (вес 1.0 каждый) = 2.0

**Расчёт**:
```
Голосов "Правда": 3.0 + 2.0 = 5.0
Всего весов: 5.0 + 2.5 = 7.5
Truth Score = 5.0 / 7.5 = 0.667 = 66.7%
```

#### Интерпретация

| Значение | Интерпретация |
|----------|---------------|
| 0-30%    | Вероятный фейк |
| 30-50%   | Сомнительно |
| 50-70%   | Скорее правда |
| 70-90%   | Достоверно |
| 90-100%  | Очень достоверно |

---

### 2. Приоритет (Priority Score)

**Диапазон**: 0.00-1.00  
**Назначение**: Определить, насколько срочно нужно решить проблему

#### Формула

```
priority_score = 
    truth_score         × 35% +
    urgency_score       × 30% +
    impact_score        × 25% +
    inconvenience_score × 10%
```

#### Компоненты

##### 2.1. Truth Score (35% веса)

- **Самый важный фактор**
- Если проблема фейк (< 30%), приоритет автоматически низкий
- Нет смысла решать несуществующую проблему

##### 2.2. Urgency Score (30% веса)

- **Срочность**: насколько быстро нужно решить
- Шкала: 1-5 (нормализуется к 0.0-1.0)
- Взвешенное среднее оценок пользователей

**Примеры**:
- 1 = Может подождать месяцы
- 3 = Желательно решить в течение недели
- 5 = Критично, требует немедленного решения

##### 2.3. Impact Score (25% веса)

- **Влияние**: сколько людей затронуто
- Шкала: 1-5 (нормализуется к 0.0-1.0)
- Взвешенное среднее оценок пользователей

**Примеры**:
- 1 = Затрагивает 1-5 человек
- 3 = Затрагивает район (100-500 человек)
- 5 = Затрагивает весь город (тысячи людей)

##### 2.4. Inconvenience Score (10% веса)

- **Неудобство**: насколько это мешает повседневной жизни
- Шкала: 1-5 (нормализуется к 0.0-1.0)
- Взвешенное среднее оценок пользователей

**Примеры**:
- 1 = Незначительное неудобство
- 3 = Заметное неудобство
- 5 = Серьёзное препятствие для жизни

#### Пример расчёта

**Исходные данные**:
```
Truth Score:         85% (0.85)
Urgency:            4.2/5 → 0.84
Impact:             3.8/5 → 0.76
Inconvenience:      4.5/5 → 0.90
```

**Расчёт**:
```
priority_score = 
    0.85 × 0.35 = 0.2975
  + 0.84 × 0.30 = 0.2520
  + 0.76 × 0.25 = 0.1900
  + 0.90 × 0.10 = 0.0900
  ─────────────────────────
  = 0.8295 (82.95%)
```

#### Интерпретация

| Значение | Приоритет | Рекомендация |
|----------|-----------|--------------|
| 0.0-0.3  | Низкий    | Можно отложить |
| 0.3-0.5  | Средний   | Решить в течение месяца |
| 0.5-0.7  | Высокий   | Решить в течение недели |
| 0.7-0.9  | Очень высокий | Решить в течение 2-3 дней |
| 0.9-1.0  | Критический | Требует немедленного решения |

## 🔄 Процесс пересчёта

### Когда происходит пересчёт?

Scores пересчитываются автоматически после:
1. Создания нового голоса
2. Изменения существующего голоса
3. Удаления голоса

### Как происходит пересчёт?

1. **Получение актуальных голосов**
   ```python
   votes = db.query(Vote).filter_by(
       problem_entity_id=problem_id,
       is_current=True
   ).all()
   ```

2. **Расчёт каждого score**
   - Truth Score: взвешенная доля голосов "Правда"
   - Urgency/Impact/Inconvenience: взвешенное среднее оценок
   - Priority Score: комбинация всех компонентов

3. **Создание новой версии проблемы**
   - Старая версия помечается `is_current=False`
   - Создаётся новая версия с обновлёнными scores
   - Сохраняется причина изменения: `"scores_recalculated"`

### Event Sourcing

Каждый пересчёт создаёт новую версию проблемы:
- Полная история изменений scores
- Можно отследить, как менялся приоритет со временем
- Аудит: кто и когда повлиял на оценки

## 💡 Преимущества системы

### 1. Защита от манипуляций

- **Взвешенные голоса**: нельзя накрутить приоритет ботами
- Голос опытного пользователя весит больше
- Новые аккаунты имеют минимальное влияние

### 2. Фильтрация фейков

- Truth Score < 30% → проблема помечается как сомнительная
- Низкая достоверность автоматически снижает приоритет
- Модераторы могут фокусироваться на проверенных проблемах

### 3. Справедливое ранжирование

- Учитывается не только количество голосов, но и их качество
- Проблемы с высоким impact получают больше внимания
- Срочные проблемы поднимаются в приоритете

### 4. Прозрачность

- Все формулы открыты и понятны
- Пользователи видят, как формируется приоритет
- История изменений scores доступна в timeline

| Urgency Score | Красный (danger) | Срочность |

## 📝 Примеры сценариев

### Сценарий 1: Яма на дороге

**Голоса**:
- 10 водителей подтвердили (truth: true, urgency: 4, impact: 3, inconvenience: 4)
- 1 человек сказал фейк (truth: false)

**Результат**:
- Truth Score: ~91% (10 vs 1)
- Urgency: 4/5 = 0.8
- Impact: 3/5 = 0.6
- Inconvenience: 4/5 = 0.8
- **Priority: 0.79 (высокий приоритет)**

### Сценарий 2: Граффити на стене

**Голоса**:
- 5 человек подтвердили (truth: true, urgency: 2, impact: 1, inconvenience: 1)

**Результат**:
- Truth Score: 100%
- Urgency: 2/5 = 0.4
- Impact: 1/5 = 0.2
- Inconvenience: 1/5 = 0.2
- **Priority: 0.47 (средний приоритет)**

### Сценарий 3: Фейковая проблема

**Голоса**:
- 2 человека подтвердили
- 8 человек сказали фейк

**Результат**:
- Truth Score: 20%
- **Priority: ~0.15 (низкий приоритет, даже если urgency высокий)**

## 🔧 Техническая реализация

### Файлы

- **Backend**: `./backend/app/services/scoring.py`
- **Models**: `./backend/app/models/problem.py`, `/backend/app/models/vote.py`
- **API**: `./backend/app/api/v1/votes.py`
- **Workers**: `./backend/app/workers/tasks_scoring.py`

### Функция пересчёта

```python
def recalculate_scores(
    db: Session,
    problem_entity_id: int,
    changed_by_id: int,
) -> Problem:
    """
    Пересчитывает все *_score для проблемы.
    Создаёт новую версию с обновлёнными scores.
    """
    # Получить актуальные голоса
    votes = db.query(Vote).filter_by(
        problem_entity_id=problem_entity_id,
        is_current=True
    ).all()
    
    # Рассчитать scores
    truth_score = calculate_truth_score(votes)
    urgency_score = weighted_avg(votes, "urgency", max_val=5)
    impact_score = weighted_avg(votes, "impact", max_val=5)
    inconvenience_score = weighted_avg(votes, "inconvenience", max_val=5)
    
    priority_score = (
        truth_score * 0.35 +
        urgency_score * 0.30 +
        impact_score * 0.25 +
        inconvenience_score * 0.10
    )
    
    # Создать новую версию
    return create_new_version(
        db, Problem, problem_entity_id, changed_by_id,
        change_reason="scores_recalculated",
        truth_score=round(truth_score, 4),
        priority_score=round(priority_score, 4),
        # ...
    )
```
